# Songboard — слова песен для Holyrics

Одностраничный инструмент (чистый HTML/CSS/JS, без сборки и зависимостей),
который помогает подготовить текст песни с holychords.pro для показа в Holyrics:

1. **Поиск** — открывает `holychords.pro/search` в новой вкладке.
2. **Вставка** — вы вставляете скопированный со страницы текст (с аккордами) и ссылку на песню.
3. **Аккорды убираются автоматически**, текст раскладывается по блокам
   (Куплет/Припев/Bridge и т.д., либо по пустым строкам, если меток нет).
4. **Редактирование** — блоки можно переставлять, объединять, разбивать по позиции
   курсора, переименовывать, дублировать, удалять.
5. **Экспорт** — копирование в буфер обмена или скачивание `.txt`, который
   вставляется в редактор песни Holyrics (пустая строка = граница слайда).

Про `.mufl`: это закрытый бинарный формат сериализации (Java Object Serialization)
без публичной спецификации полей класса песни внутри Holyrics. Собрать гарантированно
совместимый бинарник без доступа к реальному образцу файла нельзя — неправильный файл
Holyrics просто откажется импортировать. Поэтому инструмент выдаёт «правильный» текст,
который вставляется в редактор Holyrics за несколько секунд и гарантированно работает.
Если у вас есть пример `.mufl`-файла, экспортированного из вашей Holyrics (Песня →
Экспорт → Групповой экспорт), его можно разобрать и добавить прямой бинарный экспорт.

Никакого бэкенда нет: страница ничего не отправляет на сервер, весь разбор текста —
в браузере пользователя.

## Деплой на GitHub Pages (без сторонних платформ)

1. Создайте новый репозиторий на GitHub, например `songboard`.
2. Положите `index.html` (этот файл) в корень репозитория и запушьте:
   ```bash
   git init
   git add index.html
   git commit -m "Songboard: лирика для Holyrics"
   git branch -M main
   git remote add origin https://github.com/<ваш-логин>/songboard.git
   git push -u origin main
   ```
3. В репозитории откройте **Settings → Pages**.
4. В разделе **Build and deployment → Source** выберите **Deploy from a branch**.
5. Ветка — `main`, папка — `/ (root)`, нажмите **Save**.
6. Через минуту-две сайт будет доступен по адресу:
   `https://<ваш-логин>.github.io/songboard/`

### Свой домен (опционально)
Если хотите использовать собственный домен вместо `github.io`:
1. В том же разделе **Settings → Pages** впишите домен в поле **Custom domain**.
2. У вашего DNS-провайдера добавьте `CNAME`-запись, указывающую на
   `<ваш-логин>.github.io` (для поддомена) либо `A`-записи на IP GitHub Pages
   (для корневого домена — см. официальную документацию GitHub Pages).
3. Подождите обновления DNS и включите **Enforce HTTPS** в настройках Pages.

Это полностью статический хостинг от GitHub — никаких Vercel/Netlify/прочих
посредников не требуется.
